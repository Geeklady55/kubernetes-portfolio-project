Project Highlights

Container Deployment

Deploys a containerized web application into a local Kubernetes cluster using Docker and kind.

Reliability

Includes readiness and liveness probes to support self-healing and availability.

Observability

Demonstrates logs, metrics, events, and operational troubleshooting workflows.

Scaling

Supports replica scaling and horizontal autoscaling concepts for production readiness.

Architecture Overview

User Browser → Service → Deployment → Pods → Container

  • Service provides stable access to the application
  • Deployment maintains the desired number of replicas
  • Pods run the application workload
  • Container serves the web content

Operational Focus

Logs

Used to inspect application activity and container errors.

kubectl logs <pod-name>

Metrics

Used to review CPU and memory utilization.

kubectl top pods

Events

Used to identify scheduling, restart, and deployment issues.

kubectl get events

Health Checks

Readiness and liveness probes validate application health and recovery behavior.

Incident Example

A local API connectivity issue prevented deployment validation. Investigation included Docker health, cluster status, kubectl context, and service recovery steps.

Failure Simulation

This project can demonstrate self-healing by deleting a pod and watching Kubernetes recreate it automatically.
